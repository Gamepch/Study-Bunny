# 관리자 인증 보안 개선사항

## 문제점 (개선 전)
1. **`is_admin()` 함수의 취약성**: 단순히 `username == 'admin'`만 확인
2. **클라이언트 username 변조**: 쿼리 파라미터나 요청 본문에서 받은 username을 신뢰
   - `/api/admin/users?username=admin` 으로 변조 가능
   - `/api/admin/reports?username=admin` 으로 변조 가능
3. **세션 미활용**: Flask 세션이 설정되어 있지만 인증에 사용하지 않음

---

## 구현된 보안 개선사항

### 1. 서버 세션 기반 인증 (Session-based Authentication)
- **원리**: 사용자 로그인 시 서버 세션에 사용자 정보를 저장
- **장점**: 클라이언트에서 세션 값을 변조할 수 없음 (서버 메모리에 저장)
- **보안성**: 높음

#### 구현 코드
```python
def get_current_user():
    """세션에서 인증된 사용자를 조회"""
    return session.get('username')

def require_admin(f):
    """관리자 권한이 필요한 엔드포인트를 위한 데코레이터"""
    from functools import wraps
    @wraps(f)
    def decorated_function(*args, **kwargs):
        username = get_current_user()
        if not username or not is_admin(username):
            return jsonify({"message": "fail", "reason": "관리자 권한이 필요합니다."}), 403
        return f(*args, **kwargs)
    return decorated_function
```

### 2. 개선된 `is_admin()` 함수
- **이전**: `username == 'admin'`만 체크
- **변경**: 데이터베이스에서 username이 'admin'인지 확인 (향후 is_admin 칼럼 추가 가능)
- **보안성**: 중간 (DB 기반이지만, 현재는 여전히 username == 'admin')

### 3. 로그인 시 세션 저장
모든 로그인 엔드포인트에서 세션에 사용자 정보를 저장:
- `/api/login` (로컬 로그인)
- `/auth/google/callback` (구글 로그인)
- `/auth/google/complete-signup` (구글 회원가입)

```python
# 로그인 성공 시
session['username'] = user['username']
session['nickname'] = user['nickname']
session['profile_url'] = user['profile_url']
session['login_method'] = user['login_method']
session.permanent = False
```

### 4. 관리자 API에 `@require_admin` 데코레이터 적용
다음 엔드포인트들에 적용:
- `GET /api/admin/users` - 전체 사용자 목록 조회
- `GET /api/admin/reports` - 신고 목록 조회
- `PUT /api/admin/reports/<id>` - 신고 상태 업데이트

**변경 전**:
```python
@app.route('/api/admin/users', methods=['GET'])
def get_all_users():
    username = request.args.get('username')  # ⚠️ 취약점: 쿼리 파라미터에서 받음
    if not username or not is_admin(username):
        return jsonify({"message": "fail", ...}), 403
```

**변경 후**:
```python
@app.route('/api/admin/users', methods=['GET'])
@require_admin  # ✅ 세션에서 자동으로 인증 확인
def get_all_users():
    # username을 요청에서 받을 필요 없음
    # 세션에 저장된 인증된 사용자만 접근 가능
```

### 5. 새로운 인증 관련 엔드포인트

#### `/api/logout` (POST)
```python
@app.route('/api/logout', methods=['POST'])
def logout():
    """사용자 로그아웃 - 세션 초기화"""
    username = get_current_user()
    if username:
        session.clear()
        return jsonify({"message": "success", "reason": "로그아웃되었습니다."}), 200
```

#### `/api/auth/check` (GET)
```python
@app.route('/api/auth/check', methods=['GET'])
def check_auth():
    """현재 인증 상태 및 사용자 정보 확인"""
    username = get_current_user()
    if username:
        # DB에서 최신 사용자 정보 조회
        return jsonify({
            "authenticated": True,
            "user": {
                "username": user['username'],
                "nickname": user['nickname'],
                "is_admin": is_admin(user['username'])
            }
        }), 200
    return jsonify({"authenticated": False}), 200
```

## 클라이언트 코드 변경 사항

### API 호출 변경

**변경 전** (취약한 코드):
```javascript
// 관리자 사용자 목록 조회
fetch('/api/admin/users?username=admin')  // ⚠️ username을 파라미터로 전달
  .then(response => response.json())
```

**변경 후** (안전한 코드):
```javascript
// 관리자 사용자 목록 조회
fetch('/api/admin/users')  // ✅ 서버가 세션에서 사용자 확인
  .then(response => response.json())
```

### 로그아웃 구현
```javascript
async function logout() {
  const response = await fetch('/api/logout', {
    method: 'POST',
    credentials: 'include'  // 중요: 쿠키/세션 포함
  });
  
  const data = await response.json();
  if (data.message === 'success') {
    // 로그아웃 성공 - UI 업데이트
    window.location.href = '/login';
  }
}
```

### 인증 상태 확인
```javascript
async function checkAuthentication() {
  const response = await fetch('/api/auth/check');
  const data = await response.json();
  
  if (data.authenticated) {
    console.log('인증된 사용자:', data.user);
    if (data.user.is_admin) {
      // 관리자 UI 표시
    }
  } else {
    // 로그인 페이지로 리다이렉트
    window.location.href = '/login';
  }
}
```

## 추가 권장사항

### 1. 다른 엔드포인트에도 적용
현재는 관리자 인증만 개선했습니다. 다음 엔드포인트들도 동일한 취약점을 가지고 있으므로 개선 권장:

- `PUT /api/posts/<id>` - 게시물 수정 (현재 요청 본문에서 username 받음)
- `DELETE /api/posts/<id>` - 게시물 삭제 (현재 요청 본문에서 username 받음)
- `POST /api/posts/<id>/like` - 좋아요 (현재 요청 본문에서 username 받음)
- `PUT /api/comments/<id>` - 댓글 수정 (현재 요청 본문에서 username 받음)
- `DELETE /api/comments/<id>` - 댓글 삭제 (현재 요청 본문에서 username 받음)

### 2. JWT 토큰 기반 인증 (선택사항)
더 강화된 보안을 원한다면 JWT 토큰 기반 인증 추가:
```python
from flask_jwt_extended import JWTManager, create_access_token, jwt_required

# 로그인 시 JWT 토큰 발급
access_token = create_access_token(identity=user['username'])

# 보호된 엔드포인트에서 JWT 검증
@app.route('/api/admin/users', methods=['GET'])
@jwt_required()
def get_all_users():
    current_user = get_jwt_identity()
    ...
```

### 3. CSRF 보호
세션 기반 인증에서는 CSRF 공격 방지 필요:
```python
from flask_wtf.csrf import CSRFProtect

csrf = CSRFProtect(app)

# POST, PUT, DELETE 요청 시 CSRF 토큰 포함
```

### 4. 관리자 권한 추가 필드
데이터베이스에 `is_admin` 칼럼 추가하여 더 안전한 관리:
```sql
ALTER TABLE users ADD COLUMN is_admin INTEGER DEFAULT 0;
```

## 테스트 방법

### 1. 관리자 API 접근 테스트
```bash
# 세션 없이 접근 (실패해야 함)
curl -X GET http://localhost:5000/api/admin/users

# 세션과 함께 접근 (성공해야 함)
curl -X GET http://localhost:5000/api/admin/users \
  -H "Cookie: session=..."
```

### 2. 로그인/로그아웃 테스트
```bash
# 로그인
curl -X POST http://localhost:5000/api/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "..."}'

# 로그아웃
curl -X POST http://localhost:5000/api/logout \
  -H "Cookie: session=..."
```

### 3. 인증 상태 확인
```bash
curl -X GET http://localhost:5000/api/auth/check \
  -H "Cookie: session=..."
```
